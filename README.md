# germantessmer.github.io

Personal academic website for Germán A. Tessmer, built with [Quarto](https://quarto.org) and deployed via GitHub Pages.

🌐 **Live site:** https://germantessmer.github.io/

---

## Stack

- **Framework:** Quarto 1.6.33
- **Language:** R 4.5.2
- **Deploy:** GitHub Pages → `main` branch → `/docs` folder

## Site structure

| File | Content |
|------|---------|
| `index.qmd` | About / bio |
| `research.qmd` | Working papers, data, publications |
| `eph.qmd` | EPH Ecosystem — all EPH-related work (research, documentation, datasets, reports) |
| `scripts/` | Public R scripts (`bridge_eph.R`, `separa_eph.R`), copied from `C:\oes\eph_full\dataverse\scripts_publicos` |
| `teaching.qmd` | Teaching history |
| `cv.qmd` | Curriculum vitae |
| `styles.css` | Custom styles |
| `profile.jpg` | Profile photo |
| `robots.txt` | Search engine crawl rules + sitemap pointer |

## Design

Logo palette: dark blue `#063F5D` · steel gray `#5C7381` · background gray `#F2F2EF`.

Text hierarchy in `styles.css` (no pure black): headings `#193644` · bold text `#5B6274` · body `#2c2c2c`.

The **EPH Ecosystem** navbar item sits last (right of CV) and is highlighted with inverted colors (gray-blue background `#5B6274`, light text `#F2F2EF`) via `styles.css`.

## SEO & Analytics

- **Google Analytics (GA4):** `G-5VY12JPMM3` — configured in `_quarto.yml`
- **Google Search Console:** verified via HTML meta tag (URL Prefix method). Sitemap auto-generated by Quarto at `/docs/sitemap.xml`
- **Open Graph / Twitter Card:** configured in `_quarto.yml` — enables rich previews when shared on LinkedIn, Facebook, X, etc.
- **Meta descriptions:** each `.qmd` file has a `description` field in its YAML frontmatter for Google search results

> **Note:** GitHub Pages sites must use the **"URL Prefix"** property type in Search Console (not "Domain"), since DNS is managed by GitHub.

## Update workflow

```bash
# 1. In RStudio: Build → Render Website (or: quarto render)

# 2. In Git Bash
cd /c/germantessmer.github.io
git add .
git commit -m "describe change"
git push
```

> **Always render before pushing.** GitHub Pages serves the `/docs` folder generated by Quarto.
